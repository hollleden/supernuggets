# supernuggets

A personal "second brain" system. A Telegram bot (`@supernuggets_bot`) ingests anything the user sends it (text, photos, videos, image galleries, social media URLs). It uses AI to download, transcribe, summarize, fact-check, tag, classify, and saves everything to Supabase. This web app (`~/supernuggets`) is the visual front door — a Pinterest-style grid for browsing, filtering, editing, and rediscovering saved content.

The user is **non-technical**. Explain in plain language before running anything. Confirm before destructive actions. Heavy lifting is yours; user pastes credentials, clicks approve, occasionally clicks a vendor dashboard button.

## Bot output format (locked — receipt-style HTML, no emoji)

Aesthetic: technical blueprint / retro computer manual / digital zine. Stark, schematic, ASCII-leaning. No picture emojis anywhere (replaced with bracketed text tags). Unicode geometric symbols (▪ • ✓ ✗ ⌫ ⬈) are kept.

```
[BEAUTY] RETINOIDS VS PDRN
--------------------
SUMMARY
▪ bullet 1
▪ bullet 2
▪ bullet 3
--------------------
MENTIONED
• <a href="google-search-url">Brand — Product</a>
--------------------
FACT-CHECK
✓ <a href="google-search-url">TERM</a> · one-sentence reasoning
--------------------
TRANSCRIPT
<pre>raw text preserved here</pre>
--------------------
TAGS
#Folder #tag_with_underscore #another_tag
```

Inline buttons under each message: `⬈ OPEN` (https URLs only — hidden during localhost dev) and `⌫ DELETE` (with a confirmation step: `✓ YES DELETE / ✗ CANCEL`).

Bot status replies use bracketed text tags, never emojis:
- `[err]`, `[warn]`, `[limit]`, `[batch]`, `[deleted]`, `[cancelled]`, `[denied]`, `[unsupported]`
- Real errors (not `[limit]`) get a `ping @holeden` footer

**Rules baked into `pipeline.py`:**
- Title: `[FOLDER]` prefix (UPPERCASE, in brackets) + UPPERCASE title, one line, no subtitle. No emoji.
- HTML: only `<a>` and `<pre>` (no `<b>`, no decorative tags)
- Telegram entities: only `&amp; &lt; &gt; &quot;` supported — `&#x27;` for `'` is REJECTED. Use `html.escape(s, quote=False)`.
- Telegram message cap: 4096 chars. Transcript auto-truncates with `… (truncated — full text on web)` to fit.
- Empty sections (MENTIONED, FACT-CHECK) are dropped entirely.
- TRANSCRIPT is always present (raw input preserved). For image entries: prepends `CAPTION:` (if any) + `OCR:` (Claude-extracted). No visual description.
- Tags: 3-5, lowercase, underscores not hyphens (`#anti_aging` not `#anti-aging`). Folder is added as the lead hashtag by code, not by Claude (prevents duplication).
- Mentioned/Fact-check links: Google search URL wrapped invisibly in the term name. Web search tool (Anthropic's built-in) deferred until usage warrants the cost.

- `ADMIN_ID` (in `.env`) = unlimited
- Everyone else = 5 entries/day, resets at 00:00 UTC (truly UTC — `database.py` uses `datetime.now(timezone.utc).date()`)
- Hit by counting today's `entries` rows for that `user_id` via `database.get_today_count`
- Quota message shows dynamic time remaining: `[limit] 5/5 today, resets in 6h 23m`

## Gotchas you'd otherwise re-discover

- **Telegram HTML parser** only accepts named entities `&amp; &lt; &gt; &quot;`. Numeric like `&#x27;` (apostrophe) gets the whole message rejected. Always escape with `html.escape(s, quote=False)`.
- **Telegram inline button URLs** can't be `http://localhost:*` — gets rejected as "Wrong HTTP URL". Only show OPEN when `WEB_URL.startswith("https://")`.
- **Telegram message limit**: 4096 chars including HTML tags. We reserve 96 for safety (`TG_MAX_LEN = 4000`).
- **Anthropic prompt caching** requires the system prompt to be wrapped: `system=[{"type": "text", "text": SYS, "cache_control": {"type": "ephemeral"}}]`.
- **TextEdit on Mac autosaves to iCloud** — caused us to think `.env` was saved when it wasn't. Use `setup_env.sh` or `nano`, not TextEdit, for env files.
- **`dotenv.load_dotenv()` must be called BEFORE `import database` / `import pipeline`** — those read env at module import time. Order in `bot.py` is important.
- **Haiku tool use needs strict schemas** — Haiku 4.5 leaks XML-style `<parameter name="...">` tags into JSON array fields if the tool schema is loose. Always set `additionalProperties: false` on objects and `minItems`/`maxItems` on arrays. Sonnet doesn't have this issue.
- **Telegram album items arrive as separate messages** with a shared `media_group_id`. Use a debounce buffer (1.5s) keyed by group id so albums become one entry. Captions only attach to the FIRST item.
- **Pillow opens HEIC/PNG/GIF/WebP/JPEG** but JPEG can't carry alpha — flatten RGBA/LA/P onto white before re-encoding, else PIL raises.
- **`load_dotenv()` without args sometimes doesn't override** if the env var is already set to empty. Use `load_dotenv(".env", override=True)` in test scripts.
